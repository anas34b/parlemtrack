# BUG-005 — URL du CSV Datan horodatée, cassée entre deux runs

- Symptôme : un run réel du pipeline lancé quelques minutes après le précédent a échoué à télécharger le CSV des scores Datan (`404 Not Found`), alors que l'URL avait fonctionné juste avant.
- Cause racine : l'URL directe du fichier CSV sur `static.data.gouv.fr` contient un horodatage de publication dans le chemin (`.../20260720-211401/deputes-active.csv`) qui change à chaque republication du dataset (hebdomadaire selon la mission, mais observé ici à quelques minutes d'intervalle lors des tests). Coder cette URL en dur dans `pipeline/collect/datan.py` la rendait obsolète dès la publication suivante.
- Correction : `_resoudre_url_csv()` interroge désormais l'API stable du dataset (`https://www.data.gouv.fr/api/1/datasets/<id>/`) pour récupérer l'URL courante de la ressource CSV avant chaque téléchargement, au lieu de dépendre d'une URL figée. Par ailleurs, cette panne a révélé que la récupération Datan n'était pas isolée du run principal (`pipeline/run.py`) : une erreur ici faisait échouer et annuler (rollback) toute la collecte scrutins/députés déjà réussie. `mettre_a_jour_scores` est désormais dans son propre bloc `try/except`, commité séparément, avec un `logger.warning` en cas d'échec — cohérent avec la description du job Datan comme "job séparé" dans MISSION_BC02.md.
- Test de régression ajouté : `tests/pipeline/test_datan.py::test_recuperer_scores_datan_resout_url_puis_parse_le_csv`.
- Commit : à renseigner par l'utilisateur lors du commit correctif.
