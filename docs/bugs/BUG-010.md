# BUG-010 — module `pipeline` absent de l'image Docker du backend

- Symptôme : au premier `docker compose -f docker-compose.prod.yml up --build`, le conteneur `backend` démarrait puis crashait en boucle (`Restarting`), avec `ModuleNotFoundError: No module named 'pipeline'`.
- Cause racine : `backend/Dockerfile` ne copiait que `backend/`, `alembic/` et `alembic.ini` dans l'image de runtime. Or `backend/app/services/actualites_service.py` importe `pipeline.store.actualites_cache` — le backend dépend réellement du package `pipeline/` (code partagé entre l'API et le pipeline de collecte), qui n'avait pas été identifié comme nécessaire au runtime de l'API.
- Correction : ajout de `COPY pipeline/ pipeline/` dans le stage `runtime` de `backend/Dockerfile`, aux côtés de `backend/`.
- Test de régression ajouté : aucun test unitaire (défaut de configuration Docker, pas de code métier) — vérifié par ré-exécution réelle de `docker compose up --build` : le conteneur `backend` démarre et reste stable, `GET /api/health` répond `200`.
- Commit : à renseigner par l'utilisateur lors du commit correctif.
