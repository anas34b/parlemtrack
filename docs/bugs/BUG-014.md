# BUG-014 — flux RSS Le Figaro parfois illisible (XML malformé)

- Symptôme : lors de certains runs du pipeline, le flux RSS `https://www.lefigaro.fr/rss/figaro_politique.xml` échoue à être parsé, avec une erreur `feedparser` de type "mismatched tag" — le flux Le Figaro n'apparaît alors pas dans les actualités collectées pour ce run.
- Cause racine : le flux XML publié par Le Figaro est occasionnellement malformé côté source (balise mal fermée) — ce n'est pas un défaut du code de collecte, mais une donnée externe non fiable, hors du contrôle du projet.
- Décision : **ne pas corriger** — `pipeline/collect/rss_presse.py::recuperer_actualites` détecte déjà ce cas via `flux.bozo` (attribut `feedparser` signalant un flux malformé) et le journalise en `WARNING` (`flux RSS %s illisible : %s`) sans faire planter le pipeline ni les autres flux. Documenté ici comme limite connue et acceptée, dépendante d'une source externe, plutôt que comme un bug à corriger.
- Test de régression ajouté : couvert par le test existant sur un flux XML malformé dans `tests/pipeline/test_rss_presse.py` (vérifie que `bozo` déclenche bien un `WARNING` et n'interrompt pas la collecte des autres sources) — aucun nouveau test nécessaire.
- Commit : à renseigner par l'utilisateur lors du commit correctif.
