# BUG-006 — lien_an construit avec l'uid au lieu du numéro de scrutin

- Symptôme : détecté par **contrôle manuel** d'un scrutin comparé au site source (pas par les tests automatiques) — le lien `lien_an` stocké en base pour ouvrir un scrutin sur le site de l'Assemblée nationale renvoyait une erreur 404. Exemple : `https://www.assemblee-nationale.fr/dyn/17/scrutins/VTANR5L17V8418` (404) au lieu de `https://www.assemblee-nationale.fr/dyn/17/scrutins/8418` (200).
- Cause racine : `pipeline/parse/scrutins.py::parse_scrutin` construisait l'URL avec `data["uid"]` (identifiant technique du format open data AN, ex. `VTANR5L17V8418`), en supposant par erreur que le site public utilisait le même identifiant. Le site `assemblee-nationale.fr` adresse en réalité un scrutin par son **numéro** (`8418`), pas par son uid.
- **Leçon retenue** : les tests unitaires du parser (`tests/pipeline/test_parse_scrutins.py`) validaient la présence et le format de `lien_an` (une URL bien formée contenant l'identifiant attendu), mais aucun test ne vérifiait que ce lien pointait réellement vers une page existante du site — ce n'est pas le rôle d'un test unitaire, qui travaille sur des fixtures locales sans réseau. Seule une vérification manuelle contre la source réelle a permis de détecter l'écart entre "URL syntaxiquement correcte" et "URL fonctionnellement correcte". C'est précisément la justification du **cahier de recettes** (`docs/recette.md`, Phase 6) en complément des tests automatisés : certains cas (liens externes, rendu visuel, cohérence avec la source AN) ne peuvent être validés que par une recette exécutée manuellement contre le système réel.
- Correction :
  - `pipeline/parse/scrutins.py` construit désormais `lien_an` avec `numero` (`f"https://www.assemblee-nationale.fr/dyn/17/scrutins/{numero}"`) au lieu de `uid`.
  - Les 8418 lignes déjà en base ont été corrigées par un `UPDATE` ciblé sur les liens au format `VTANR%` (pas de retéléchargement ni de reparsing complet, puisque seule la construction de l'URL était en cause).
- Test de régression ajouté : `tests/pipeline/test_parse_scrutins.py::test_scrutin_lien_an_construit_avec_le_numero_pas_luid`.
- Vérification : `https://www.assemblee-nationale.fr/dyn/17/scrutins/8418` (scrutin le plus récent en base après correction) retourne HTTP 200.
- Commit : à renseigner par l'utilisateur lors du commit correctif.
