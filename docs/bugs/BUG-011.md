# BUG-011 — écriture refusée dans `.next/cache` pour l'utilisateur non-root du frontend

- Symptôme : dans le conteneur `frontend` (image de production), les pages dynamiques se chargeaient (HTTP 200) mais les logs affichaient en boucle `Failed to update prerender cache ... Error: EACCES: permission denied, mkdir '/app/.next/cache'`.
- Cause racine : `frontend/Dockerfile` copie `.next/standalone`, `.next/static` et `public` depuis le stage `builder` vers le stage `runtime` sans préciser de propriétaire — les fichiers restent possédés par `root` (utilisateur du `COPY`). Le conteneur bascule ensuite sur l'utilisateur non-root `parlemtrack` (`USER parlemtrack`) avant de lancer `server.js`, qui tente d'écrire son cache de rendu incrémental dans `.next/cache` : ce répertoire n'existe pas encore et `parlemtrack` n'a pas le droit de le créer dans une arborescence possédée par `root`.
- Correction : dans `frontend/Dockerfile`, ajout de `--chown=parlemtrack:parlemtrack` sur les trois `COPY` du stage `runtime`, et création explicite de `.next/cache` avec les bons droits (`RUN mkdir -p .next/cache && chown parlemtrack:parlemtrack .next/cache`) avant le `USER parlemtrack`.
- Test de régression ajouté : aucun test unitaire (défaut de configuration Docker) — vérifié par ré-exécution réelle de `docker compose up --build` : plus aucune erreur `EACCES` dans les logs du conteneur `frontend`.
- Commit : à renseigner par l'utilisateur lors du commit correctif.
