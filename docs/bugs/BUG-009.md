# BUG-009 — rate limit Mistral (429) en rafale lors d'un lot de génération

- Symptôme : lors du premier lancement du lot de génération d'analyses IA sur ~360 scrutins, de nombreux appels consécutifs à l'API Mistral échouaient avec `429 Too Many Requests`, y compris après les 2 retries prévus (backoff 2s/4s) — le lot a dû être interrompu manuellement après une vingtaine d'échecs consécutifs.
- Cause racine : `pipeline/ia/mistral_client.py::generer_analyse` enchaînait les appels à l'API sans aucune pause entre deux scrutins traités par `generer_lot`, et le backoff de retry (2s puis 4s) était trop court pour laisser le temps à la fenêtre de rate limit de Mistral de se libérer — chaque tentative de retry se heurtait au même mur.
- Correction :
  - `pipeline/ia/mistral_client.py` : détection spécifique d'une erreur 429 (`_est_rate_limit`, basée sur `raw_response.status_code`) déclenchant un backoff dédié plus long (`BACKOFF_RATE_LIMIT_S = 15.0`) au lieu du backoff linéaire générique.
  - `pipeline/ia/generer_analyses.py::generer_lot` : ajout d'une pause fixe de 1,5 s (`PAUSE_ENTRE_APPELS_S`) entre deux scrutins traités, pour rester sous la limite en régime normal plutôt que de compter uniquement sur les retries en cas de dépassement.
- Test de régression ajouté : aucun test unitaire dédié (comportement dépendant du rythme réel de l'API externe, non reproductible de façon fiable en mock) — vérifié par ré-exécution réelle du lot après correction, sans nouvelle rafale de 429.
- Commit : à renseigner par l'utilisateur lors du commit correctif.
