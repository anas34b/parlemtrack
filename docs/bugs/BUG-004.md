# BUG-004 — `objet.dossierLegislatif` parfois objet, parfois chaîne

- Symptôme : lors du run réel du pipeline, l'insertion d'un scrutin échouait avec `psycopg2.ProgrammingError: can't adapt type 'dict'` sur la colonne `dossier_ref` (de type texte).
- Cause racine : le champ `scrutin.objet.dossierLegislatif` du format AN n'est pas toujours une chaîne (référence brute) : sur de nombreux scrutins récents, c'est un objet `{"libelle": ..., "dossierRef": "DLR5L17N..."}`. `pipeline/parse/scrutins.py` lisait directement la valeur brute sans normaliser ce cas, contrairement aux autres pièges déjà couverts par des fixtures (objet/tableau, chaînes numériques, etc.).
- Correction : ajout de `_extract_dossier_ref()` dans `pipeline/parse/scrutins.py`, qui retourne `dossier.get("dossierRef")` si le champ est un objet, sinon la valeur telle quelle.
- Test de régression ajouté : `tests/pipeline/test_parse_scrutins.py::test_scrutin_dossier_legislatif_objet_donne_la_reference` (fixture `tests/fixtures/scrutin_dossier_objet.json`).
- Commit : à renseigner par l'utilisateur lors du commit correctif.
