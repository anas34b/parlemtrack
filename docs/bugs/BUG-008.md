# BUG-008 — CI rouge : `ModuleNotFoundError: No module named 'backend'`

- Symptôme : le run #2 de `ci.yml` (job `tests-backend`) échouait dès la collecte des tests avec `ModuleNotFoundError: No module named 'backend'` sur `tests/pipeline/conftest.py:13`, alors que la même commande (`pytest`) fonctionnait sans problème en local.
- Cause racine : `tests/__init__.py` (créé en Phase 0 comme placeholder d'arborescence) n'avait jamais été ajouté à git — il restait sur le disque local sans être commité. Or le mode d'import par défaut de pytest ("prepend") détermine où insérer la racine du projet dans `sys.path` en remontant l'arborescence tant que des `__init__.py` sont présents : localement, `tests/__init__.py` existait sur le disque et pytest remontait jusqu'à la racine du dépôt (rendant `backend` importable). En CI, sur un checkout propre, ce fichier était absent : pytest s'arrêtait à `tests/` et insérait ce seul dossier dans `sys.path`, sous lequel `backend` (à la racine du dépôt) n'est pas trouvable — d'où l'écart 100% local / CI, jamais détecté avant le premier vrai push de tests avec dépendances inter-packages.
- Correction :
  - Ajout de `pythonpath = ["."]` dans `[tool.pytest.ini_options]` (`pyproject.toml`) : force explicitement la racine du dépôt dans `sys.path`, quelle que soit la présence ou l'absence des `__init__.py` de `tests/`. Rend ce type de divergence local/CI structurellement impossible à l'avenir.
  - `tests/__init__.py` et `tests/backend/__init__.py` ajoutés au suivi git (ils étaient restés non commités par erreur, en plus d'être la cause du bug).
  - `ci.yml` (job `tests-backend`) complété avec des conteneurs de service PostgreSQL 16 et Redis 7 (healthchecks inclus) et les variables d'environnement `DATABASE_URL`/`REDIS_URL` correspondantes : une fois le problème d'import résolu, la suite de tests échouait de toute façon faute de base de données réelle disponible en CI (nos tests `tests/pipeline` et `tests/backend` nécessitent une vraie connexion Postgres/Redis, contrairement au test placeholder initial de la Phase 0). Cette étape, prévue pour la Phase 5 dans MISSION_BC02.md, a été anticipée ici car sans elle la CI restait rouge pour toute la suite du projet.
- Test de régression ajouté : aucun test unitaire (problème de configuration d'infrastructure CI) — vérifié par simulation locale du comportement CI (suppression temporaire de `tests/__init__.py` et `.env`, `pytest --collect-only` confirmant l'absence d'erreur d'import après le correctif) et par validation syntaxique du YAML (`yaml.safe_load`).
- Commit : à renseigner par l'utilisateur lors du commit correctif.
