# BUG-018 — barre de vote par groupe écrasée en dessous de 721px

- Symptôme : observé librement pendant la recette (23/07/2026, hors grille de cas), confirmé par capture d'écran (`scrutin_mobile_ecrase.png`, viewport 375×812) — sur le détail d'un scrutin, la ligne "vote par groupe" affiche des noms de groupe tronqués (ex. "Droite Républic...", "Union des droit...") et la barre segmentée elle-même devient très étroite, peu lisible.
- Cause racine : confirmé dans `frontend/src/components/VoteParGroupeBarre.tsx` — la ligne est un flex row à 3 colonnes : le libellé du groupe a une largeur fixe `width: 140` (`flexShrink: 0`), le récapitulatif chiffré à droite une largeur fixe `width: 130` (`flexShrink: 0`), et seule la barre au centre (`flex: 1`) est flexible. Sur un viewport de 375px, 140 + 130 = 270px de largeur fixe (plus les `gap` et le padding du conteneur) ne laissent quasiment plus de place à la barre, et le nom de groupe est tronqué par `textOverflow: "ellipsis"` dans une colonne trop étroite pour l'afficher en entier — le composant n'a pas de layout adapté aux petits écrans (pas de `@media`, pas d'empilement vertical en dessous d'un certain seuil).
- Correction : **non corrigée à ce stade**. Piste : sous un seuil de largeur (ex. `max-width: 480px` ou via une classe CSS + media query comme pour `NavPrincipale`), empiler verticalement libellé / barre / récapitulatif au lieu de les mettre en ligne, ou réduire les largeurs fixes et autoriser le retour à la ligne du nom de groupe plutôt que de le tronquer.
- Test de régression ajouté : aucun pour l'instant (défaut visuel, difficile à couvrir par un test Jest sans test de rendu visuel/snapshot dédié — à envisager avec la correction, ex. Playwright avec capture d'écran de référence).
- Commit : à renseigner par l'utilisateur lors du commit correctif.
